1. 产品简介

IEC 60870-5-104 Slave Simulator 是一款基于 ASP.NET Core .NET 8 开发的工业级 IEC 60870-5-104（IEC 104）从站仿真工具，完整实现了 IEC 104 协议规范的从站（Slave/Server）侧逻辑， 可用于主站软件的调试、验收测试、协议一致性测试及教学演示。

系统通过嵌入式 Web 服务器对外提供全功能浏览器操作界面，无需安装任何客户端软件， 支持在同一进程内并发运行多个独立的从站实例（不同端口/CA），并可通过 Web UI 对每个实例的数据点、 品质、连接状态、通信记录进行实时监控与交互。

协议合规声明：本产品在以下方面严格遵循 IEC 60870-5-104 标准： 连接建立与保持（STARTDT/STOPDT/TESTFR）、初始化流程（M_EI_NA_1）、全站及分组召唤（C_IC_NA_1）、 电能量召唤（C_CI_NA_1）、时钟同步（C_CS_NA_1）、所有监视方向数据类型（TI=1~37）、 所有控制方向命令类型（TI=45~64）、完整 SBO 选择执行/取消流程、品质描述词（QDS）全部 5 位（OV/BL/SB/NT/IV）。

适用场景

  • 主站软件开发调试：为 SCADA/EMS/DMS 开发团队提供完整可控的 IEC 104 从站端点
  • 协议一致性验证：测试主站对各类数据类型、品质位、SBO 流程、召唤机制的处理能力
  • 系统集成测试：在实际设备不可用时，模拟真实RTU/PLC的行为
  • 协议培训教学：通过可视化通信记录深入理解 IEC 104 协议帧结构
  • TLS 安全测试：验证主站对加密通信的支持能力

2. 功能特性

🏭

多实例并发

同一进程内运行多个独立从站，各占不同端口和公共地址（CA）

📊

全类型数据点

支持 IEC 104 全部监视方向类型（TI 1~37，含带/不带时标系列）

⚙️

全类型遥控命令

支持全部控制方向类型（TI 45~64），含完整 SBO 选择执行流程

📡

实时通信记录

每帧 ASDU 完整解析显示，含 TI/VSQ/COT/CA/IOA/数值/PN/T 字段

🔒

TLS 加密通信

支持 PKCS12 证书，可验证主站客户端证书，完全符合 IEC 104 TLS 规范

🕐

时钟同步

接收并处理 C_CS_NA_1，自动计算主从时钟偏差并应用于 TB 类型时标

🔁

周期背景扫描

可配置间隔（秒）定时上送所有监视量，COT=2，有效防止主站断连

📋

分组召唤

数据点支持分配 GroupId（1-16），响应 QOI=21~36 分组召唤请求

⚡

脉冲命令

自动处理 QU=1（短脉冲 200ms）和 QU=2（长脉冲 2000ms）自动复位

🌐

浏览器操作界面

无需安装客户端，任意浏览器访问 http://localhost:5010 即可操作

🔔

遥控返校确认

遥控命令触发弹窗确认，30秒倒计时，支持 SBO 和直接执行两种模式

🔧

APCI 参数配置

K/W/T0/T1/T2/T3 全部可配置，默认值完全符合 IEC 104 标准

4. 部署与启动

4.1 直接运行

确保已安装 .NET 8 Runtime，在项目目录执行：

# 切换到项目目录
cd d:\TestResearch\iec04\IEC104Simulator

# 开发模式运行
dotnet run

# Release 模式（推荐）
dotnet run -c Release

4.2 发布为独立可执行文件

# 发布到 publish 目录
dotnet publish -c Release -o publish --self-contained false

# 运行发布版本
.\publish\IEC104Simulator.exe

4.3 访问 Web UI

启动后打开浏览器访问：

http://localhost:5010

程序启动时会自动创建一个端口 2404、公共地址 1 的默认从站实例并立即启动监听。

防火墙注意：如需从其他主机连接，请确保 Windows 防火墙允许 TCP 入站端口 2404（或其他自定义端口）。 Web UI 端口 5010 通常只需本地访问。

4.4 配置监听接口

默认 Web UI 监听 0.0.0.0:5010（所有网卡），IEC 104 从站也监听 0.0.0.0:<Port>。 如需指定端口，可在 Program.cs 最后一行修改 app.Run("http://0.0.0.0:5010")。

5. 首次使用

1

启动程序并打开浏览器

运行 dotnet run -c Release，打开浏览器访问 http://localhost:5010。 顶部状态栏显示 ● SignalR 已连接 表示 Web 通道正常。

2

确认默认实例已运行

页面顶部的实例标签页中会看到 Port:2404 CA:1，绿色圆点表示已在监听。 点击该标签进入实例面板。

3

让主站连接

在主站软件（SCADA/测试工具）中配置目标 IP 为本机地址、端口 2404、CA=1， 建立 TCP 连接后发送 STARTDT_ACT，从站将回 STARTDT_CON 并发送 M_EI_NA_1 初始化结束帧。

4

观察连接与通信

连接栏会显示主站 IP 地址，通信记录标签页会实时展示每条 ASDU 的完整解析。 点击每行可展开 ASDU 字段详情。

5

修改数据值并触发自发上送

在数据点表格中点击 ✏️ 修改值，或点击 📡 手动触发自发上送（COT=3）。 主站即可收到更新后的遥信/遥测值。

9. 品质描述词（QDS）操作

IEC 104 中每个带品质的测量值携带一个 8 位品质描述词（QDS）。系统完整支持以下 5 个有效位：

位缩写全称含义（=1 时）
bit0OVOverflow溢出：测量值超出范围
bit4BLBlocked被闭锁：数值锁定、不跟随实际变化
bit5SBSubstituted被取代：数值由人工置入
bit6NTNonTopical非当前：数值非最新采样
bit7IVInvalid无效：数值不可信

在数据点表格中，各 QDS 列显示 False（正常）或 True（置位）。点击任意品质单元格可立即反转该位，系统通过 UpdateDataPoint Hub 方法异步更新并以 COT=3 广播给所有主站。

测试提示：可通过同时设置 IV=True 模拟传感器故障场景，测试主站对无效数据的告警处理逻辑。

13. APCI 传输层详解

IEC 104 基于 TCP/IP，在 TCP 之上定义了 APCI（应用协议控制信息）。系统使用 lib60870.NET 实现 所有 APCI 控制机制。

13.1 帧类型

帧类型标识用途
I 帧bit0=0信息帧，携带 ASDU，含发送/接收序号
S 帧0x01监视帧，无数据，仅用于确认接收到的 I 帧
U 帧 STARTDT_ACT/CON0x07/0x0B激活/确认数据传输
U 帧 STOPDT_ACT/CON0x13/0x23停止/确认数据传输
U 帧 TESTFR_ACT/CON0x43/0x83心跳测试/确认（T3 超时触发）

13.2 传输序列

建连后的完整握手流程：

TCP Connect
→
STARTDT_ACT
→

17. SBO 流程详解

完整 SELECT → EXECUTE 流程

主站                              从站
  |                                |
  |----[SELECT] COT=6, S/E=1 ----->|
  |                                |---- 弹出用户确认对话框（30s 倒计时）
  |                                |
  |<---[ACT_CON(+)] COT=7, S/E=1--|  用户点"确认"
  |                                |
  |----[EXECUTE] COT=6, S/E=0 ---->|
  |                                |---- 已有 SELECT，自动接受，更新数据值
  |<---[ACT_CON(+)] COT=7, S/E=0--|
  |<---[M_SP_NA_1] COT=3 (自发)---|  数据变化通知
  |<---[ACT_TERM] COT=10 ----------|
  |                                |
  # SELECT 超时（60s 无 EXECUTE）→ 自动取消
  # 主动取消：主站发 DEACTIVATION(COT=8) → 从站回 DEACT_CON(COT=9)

跨连接保护

若 A 连接已完成 SELECT，B 连接随后发来同 IOA 的 EXECUTE，从站立即回 ACT_CON(PN=1)（否认）， 不会触发用户确认，确保命令源头一致性。

连接断开时清理

某主站 TCP 连接断开时，所有该连接的 pending SELECT 记录自动清除， 防止孤立选择状态阻塞后续操作。

19. TLS 加密通信

系统支持基于 X.509 证书的 TLS 加密，符合 IEC 104 TLS 扩展规范（IEC 62351-3）。

19.1 配置步骤

1

准备服务端证书

需要 PKCS12 格式（.pfx / .p12）包含证书链和私钥。可通过以下命令生成自签证书：

# 使用 OpenSSL 生成自签名证书（测试用）
openssl req -x509 -newkey rsa:2048 -keyout key.pem -out cert.pem -days 365 -nodes
openssl pkcs12 -export -out server.pfx -inkey key.pem -in cert.pem -passout pass:yourpassword
2

在新建实例对话框中启用 TLS

勾选 启用 TLS 加密通信，然后：
• 点击 浏览... 从本地选择 .pfx 文件（自动上传到服务端 certs 目录）
• 或直接在路径框中输入服务端绝对路径
• 填写 PFX 密码（无密码留空）

3

（可选）配置客户端证书验证

• 填写 CA 证书路径（PEM/DER 格式）用于验证主站客户端证书
• 勾选 要求主站提供并验证客户端证书 后仅特定主站可连接

证书位置：通过浏览器上传的证书文件保存于 <程序目录>/certs/ 文件夹， 文件名不可含路径分隔符。仅允许 .pfx、.p12、.crt、.cer、.pem 格式。

19.2 安全说明

  • 服务端证书使用 MachineKeySet | PersistKeySet | Exportable 标志加载
  • 启用 CA 验证时进行完整证书链验证（ChainValidation=true）
  • PFX 密码在内存中以字符串传递，建议生产环境使用操作系统密钥存储

20. 周期背景扫描

背景扫描（COT=2 BACKGROUND_SCAN）是 IEC 104 标准规定的数据传送方式之一，从站定期向主站 发送所有监视量，以维持数据同步并防止主站因长期无数据而断开连接。

配置

在新建实例对话框中设置 周期背景扫描间隔（秒）：

  • 0（默认）：禁用，不发送周期数据
  • 大于 0：每 N 秒向所有已连接主站广播全部监视量，COT=2

推荐值： 若主站有 15 秒无数据断连机制，建议设置为 10 秒；若主站依赖 TESTFR 保活则无需开启。 T3=20 秒的 TESTFR 心跳帧可独立维持连接，周期扫描主要用于数据同步。

22. 数据分组（GroupId）

IEC 104 允许主站对数据点分组召唤（QOI=21~36，对应第 1~16 组）。通过为数据点分配 GroupId（1–16）即可响应分组召唤请求。

分组规则

GroupId含义响应 QOI
0（默认）无分组，仅响应全站总召（QOI=20）QOI=20/0
1第 1 组QOI=21 及 QOI=20（全站包含所有组）
2第 2 组QOI=22
………
16第 16 组QOI=36

默认分组分配

  • GroupId=1：IOA 1–31（单点/双点遥信）
  • GroupId=2：IOA 100–150（归一化/标度化/浮点测量值）
  • GroupId=3：IOA 160–170（累计量/电能量）
  • GroupId=0：其他所有数据点（步位置、32位串、命令等）

分组设置在数据点表格的 分组 列中以蓝色徽标显示（"—" 表示无分组）。 添加新数据点时可在对话框的 分组 ID 字段指定。
